NAME

libhidraw -- Linux Simple I/O Library: Raw HID Module  

SYNOPSIS

#include <libhidraw.h>

void HIDRAW_open(char *name, int32_t *fd, int32_t *error);

void HIDRAW_close(int32_t fd, int32_t *error);

void HIDRAW_get_name(int32_t fd, char *name, int32_t size, int32_t *error);

void HIDRAW_get_info(int32_t fd, int32_t *bustype, int32_t *vendor,
  int32_t *product, int32_t *error);

void HIDRAW_send(int32_t fd, void *buf, int32_t size, int32_t *error);

void HIDRAW_receive(int32_t fd, void *buf, int32_t *size, int32_t *error);
Link with -lsimpleio.  

DESCRIPTION

All functions return either 0 (upon success) or an errno value (upon failure) in *error.

HIDRAW_open() opens a raw HID device. The device name, /dev/hidrawx, must be passed in the name parameter. Upon success, a file descriptor for the raw HID device is retured in *fd.

HIDRAW_close() closes a previously opened raw HID device.

HIDRAW_get_name() fetches an information string from the raw HID device. The destination buffer address must be passed in *name and the destination buffer size must be passed in size.

HIDRAW_get_info() fetches the bus type, vendor ID and product ID from the raw HID device. The bus type is returned in *bustype, and may be BUS_USB, BUS_HIL, BUS_BLUETOOTH or BUS_VIRTUAL. These values are defined in /usr/include/linux/input.h. The vendor and product ID's are returned in *vendor and *product respectively.

HIDRAW_send() sends a message (also known as a HID report) to the raw HID device. The address of the message buffer is passed in buf and the size of the message buffer is passed in size. The message size will typically be either 64 or 65 bytes, depending on whether the particular raw HID device uses the first byte for the report number.

HIDRAW_receive() receives a message (HID report) from the raw HID device. The destination address is passed in buf and the size of the message buffer is passed in *size. The message size will typically be either 64 or 65 bytes, depending on whether the particular raw HID device uses the first byte for the report number. Upon success, the number of bytes received will be returned in *size.
